### 4. Documentation Framework

**Templates and protocols** creating audit trails from source data to reported metrics.

**Methodology Statement Template**
- Baseline period definition and justification
- Metric selection rationale
- Data sources and collection methods
- Calculation approaches and emission factors used
- Uncertainty acknowledgment and limitations
- Exclusions and boundary decisions
- Quality control procedures
- Alignment with standards (GHG Protocol, ISO 14064, etc.)

**Source Data Log Template**
- Document each data input with: source, date obtained, responsible party, location stored
- Links from raw data to calculated results
- Change tracking (if data is corrected/updated)
- Creates complete audit trail

**Assumption Register Template**
- Documents all estimates, proxies, and assumptions used
- Justification for each assumption
- Uncertainty rating (high/medium/low confidence)
- Plans to replace assumptions with measured data
- Essential for honest baseline reporting

**Evidence Filing System**
- Folder structure for organizing supporting documents
- Naming conventions for easy retrieval
- Digital + physical organization protocols
- Retention schedules (how long to keep what)
- Preparation for external verification

**Baseline Report Template**
- Executive summary structure
- Metric presentation format (tables, graphs, narrative)
- Methodology section outline
- Limitations and uncertainties section
- Improvement plans and future commitments
- Ready-to-customize for stakeholder communication

**Delivery format:**
- 5 Word/PDF templates
- Folder structure recommendation document
- Sample filled examples showing good practice

### 5. Gap Assessment Framework

**Diagnostic tool** identifying what you can measure now vs. what requires investment, with prioritization logic for closing gaps strategically.

**Current Capability Assessment**
- Metric-by-metric evaluation: Do you have data? Is it reliable? Is it documented?
- Data quality scoring (1-5 scale: estimates → verified measurements)
- System maturity assessment (ad-hoc → routine → optimized)
- Identifies quick wins vs. long-term builds

**Gap Prioritization Matrix**
- Maps gaps against: stakeholder importance, implementation difficulty, cost, timeline
- Risk scoring (which missing data creates most stakeholder/compliance risk?)
- Opportunity scoring (which measurements unlock most value?)
- Generates ranked action list

**Staged Solution Design**
- **Immediate (Week 1-4):** What to implement with zero budget using existing resources
- **Short-term (Month 2-6):** Low-cost improvements (basic sensors, better logging protocols)
- **Long-term (Year 2-3):** Strategic investments (automated monitoring, digital platforms)
- Prevents perfect-being-enemy-of-good paralysis

**Technology Assessment Guide**
- When spreadsheets are sufficient vs. when software adds value
- Decision tree for sensor/IoT investments (monitoring energy, water, environmental conditions)
- Evaluation criteria for sustainability management platforms
- Cost-benefit framework preventing over-investment in shiny tools

**Delivery format:**
- Excel-based assessment tool with automated scoring
- Decision tree diagrams (PDF)
- Technology comparison tables
- Budget planning template

### 6. Quality Check Protocols

**Systematic error detection** ensuring your baseline data is accurate before you report it to stakeholders.

**Data Entry Validation**
- Common error patterns (unit mix-ups, decimal mistakes, transposed numbers)
- Automated range checks (flags values outside expected bounds)
- Consistency checks (do totals match sub-components?)
- Completeness verification (missing data flagging)

**Calculation Verification**
- Manual spot-checks on automated calculations
- Independent verification protocols (someone other than data enterer checks)
- Cross-metric consistency (do energy + emissions align?)
- Benchmark comparison (are your numbers plausible vs. industry ranges?)

**Temporal Consistency Review**
- Month-over-month change analysis (flags unusual spikes/drops)
- Seasonal pattern recognition (is variation explainable?)
- Year-over-year comparison protocols
- Investigation triggers (when to dig into apparent anomalies)

**Documentation Completeness Check**
- Are all data sources logged?
- Are assumptions documented?
- Can methodology be understood by external reviewer?
- Is evidence trail complete?

**Verification Readiness Scorecard**
- Self-assessment against third-party verification standards
- Gap identification before engaging verifiers
- Cost-saving by fixing issues internally first

**Delivery format:**
- Quality control checklist (Excel + PDF)
- Error pattern guide (PDF)
- Verification readiness scorecard (Excel)

## Frequently Asked Questions

**How is this different from hiring a consultant?**

Consultants provide hands-on implementation support but typically cost €3,000-10,000+ for baseline development. This toolkit gives you the same structured approach at €99, but you do the work yourself. Choose consultants if you lack internal capacity or face urgent deadlines. Choose this toolkit if you can allocate 4-6 hours/week and want to build internal capability.

**What if I don't have historical data?**

The toolkit shows you how to start measurement now with clear methodology, then use conservative estimates (with documented assumptions) for historical periods. You replace estimates with measured data over time. This approach is acceptable to auditors and buyers if methodology is transparent.

**Do I need fancy software or sensors?**

No. The toolkit is built for spreadsheet-based implementation. It shows you when software/sensors add value vs. when simpler approaches suffice. Most small-medium operations can build defensible baselines with the included Excel templates.

**How long before my systems are "audit-ready"?**

Minimum viable baseline: 12 weeks. Audit-quality maturity: 12-18 months as you refine data quality and fill gaps. The toolkit gets you to defensible baselines quickly, then provides the roadmap for continuous improvement to audit-grade rigor.

**What standards does this align with?**

The toolkit follows GHG Protocol principles (the global standard for carbon accounting) and incorporates ISO 14064 guidance. It's designed to satisfy most buyer requirements, sustainability certifications, and ESG reporting frameworks. Specific certifications may have additional requirements beyond core baselines.

**Can this scale to large/complex operations?**

The framework scales, but implementation complexity increases with operation size/complexity. Multi-site operations spanning countries may need additional customization. The toolkit works well for single-site operations up to ~50 staff or multi-site operations with centralized data management.

**What happens after I build my baseline?**

You'll have the infrastructure to maintain ongoing measurement, respond to stakeholder requests, and identify improvement opportunities. Stack 2 (Operational Efficiency) shows you how to use baseline data to find cost savings and margin improvement opportunities. Your baseline becomes operational intelligence, not just compliance data.
